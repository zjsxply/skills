# Repository Rules

- Skills in this repository must remain agent-agnostic.
- Do not mention specific agent brands, products, or vendor-targeted install flags in shared documentation, examples, metadata, or prompts.
- Use neutral wording such as `agent`, `skill-compatible agent`, or `client`.
- If a command supports optional agent-specific flags, omit them from repository-wide documentation unless a contributor is explicitly writing a private, agent-specific note outside this repository.
- Only these repository-level files require bilingual maintenance: `AGENTS.md`, `AGENTS.zh.md`, `README.md`, and `README.zh.md`.
- Skill content should stay English-only unless a future repository rule explicitly says otherwise.
- When updating shared repository docs, keep the English and Chinese versions aligned.
- Chinese version: [AGENTS.zh.md](AGENTS.zh.md)
